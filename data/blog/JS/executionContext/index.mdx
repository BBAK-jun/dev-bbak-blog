# 실행컨텍스트란

자바스크립트의 동작 원리를 담고 있는 핵심 개념이다. 실행컨텍스트를 정확히 이해한다면 스코프 기반으로 식별자와 식별자에 바인딩 된 값을 관리하는 방식과 호이스팅, 클로저, 태스크 큐와 함께 동작하는 이벤트 핸들러와 비동기 작업을 이해할 수 있다.

소스코드는 아래와 같이 4가지 타입으로 구분된다.

1. **전역 코드** : 전역 변수를 관리하기 위해 최상위 스코프인 전역 스코프가 필요하다. var 키워드로 선언된 전역변수 와 함수선언문으로 정의된 전역 객체의 프로퍼티와 메서드로 바인딩하고 참조하기 위해 전역객체와 연결되어야한다. 이를 위해 전역 코드가 평가되면 전역 실행 컨텍스트가 생성된다.
2. **함수 코드** : 함수 코드는 지역 스코프를 생성한다. 매개변수와 지역변수 arguments 객체를 관리해야하고 지역스코프와 전역스코프에서 시작하는 스코프 체이닝을 연결해야한다. 함수 코드가 평가되면 함수 실행 컨텍스트가 생성된다.
3. **eval 코드** : eval 코드는 Strict mode에서 독자적인 스코프를 생성한다. eval 코드가 평가되면 eval 실행 컨텍스트가 생성된다.
4. **모듈 코드** : 모듈 코드는 모듈별로 독립적인 모듈 스코프를 생성하게되고 모듈 코드가 평가되면 모듈 실행 컨텍스트가 생성된다.

앞서말한 4가지 타입에서 공통점이 존재한다. 소스코드에서 실행컨텍스트가 생성되는 조건은 코드가 평가되는것이다.

자바스크립트 엔진은 소스코드를 **평가**와 **실행** 과정으로 나누어 처리한다.

평가과정에는 실행컨텍스트를 생성하고 변수 함수등의 선언문만 먼저 실행하여 생성된 식별자를 **키**로 실행컨텍스트가 관리하는 **스코프(렉시컬 환경의 Environment Record)** 에 등록한다.

평가과정이 끝나게되면 선언문을 제외한 소스코드가 순차적으로 실행된다. (즉 런타임의 시작을 알린다.)<br/>
실행에 필요한 식별자의 참조를 실행 컨텍스트가 관리하는 스코프에서 검색하여 취득한다. 그리고 실행중 값의 변경등 소스코드의 실행 결과는 다시 실행 컨텍스트가 관리하는 스코프에 등록된다.

![](/static/images/JS/executionContext/1.png)

```js
var num

num = 100
```

자바스크립트 엔진은 위의 예제를 2개의 과정으로 나누어 처리하는데 먼저 선언문을 실행한다. <br/>
식별자 num은 실행컨텍스트가 관리하는 스코프(환경레코드)에 등록되고 undefined로 초기화된다.

평가 과정이 끝나면 실행과정이 시작하는데 `var num;` 선언문은 런타임 이전인 소스코드 평가과정에서 이미 실행이 완료되어 실행컨텍스트가 관리하는 스코프에 등록되었다. 따라서 `num = 100;`만 실행된다. 변수 num에 값을 할당하려면 먼저 변수가 선언된 변수인지 확인하는 작업이 필요하다. 이때 실행컨텍스트가 관리하는 스코프(환경레코드)에 변수가 등록되어있는지 확인을하고 등록되어있다면 값을 할당하고 할당 결과를 실행컨텍스트에 등록하여 관리한다.

```js
const NUM_ONE = 100
const NUM_TWO = 300

function example(num) {
  const NUM_ONE = 200
  const NUM_TWO = 400

  console.log(num + NUM_ONE + NUM_TWO) // 500 + 200 + 400
}

example(500)

console.log(NUM_ONE + NUM_TWO) // 100 + 300
```

위의 코드를 보고 생각을해보자

우선 전역코드를 평가해야한다. <br/>
전역코드 평가과정에서 선언문만 먼저 실행이된다. 따라서 변수 선언문과 함수 선언문이 먼저 실행되어 전역 변수와 전역 함수가 실행 컨텍스트가 관리하는 전역 스코프에 등록된다. var 키워드로 선언된 변수와 함수는 전역객체의 프로퍼티와 메서드가 된다.

전역코드가 평가가 된후 전역코드를 실행한다.<br/>
전역코드 평가 이후 런타임이 시작되어 전역 코드가 순차적으로 실행되기 시작한다. 이때 전역변수의 값이 할당되고 example 함수가 호출된다. 함수가 호출되면 순차적으로 실행되던 전역 코드의 실행을 일시 중단하고 코드 실행 순서를 변경후 함수 내부로 진입한다.

함수 호출로 인하여 함수 코드가 평가된다. 매개변수와 지역 변수 선언문 그리고 arguments 객체가 먼저 실행되고 함수 실행 컨텍스트가 관리하는 지역 스코프에 등록되며 this 바인딩도 결정된다.

함수 코드가 평가가 완료된후 런타임이 시작되어 함수 코드가 순차적으로 실행되기 시작한다. 이때 매개변수, 지역변수에 값이 할당되고 console.log 메서드가 호출된다.

이처럼 코드가 실행되려면 스코프를 구분하여 식별자와 바인딩된 값이 관리되어야한다. 그리고 중첩 관계에의해 스코프 체인을 형성하여 식별자를 검색할 수 있어야하며, 전역객체의 프로퍼티도 전역변수마냥 검색할 수 있어야한다.

따라서 스코프 식별자 코드 실행순서 등의 관리가 필요하다. <br/>
이를 **실행컨텍스트** 가 관리한다. 실행컨텍스트의 책임과 역할은 아래와 같다.

- 선언에 의해 생성된 모든 식별자를 스코프를 구분하여 등록하고 상태변화를 지속적으로 트래킹할 수 있어야한다.
- 스코프는 중첩 관계에 의해 스코프 체인을 형성해야하며 이를 통해 상위 스코프로 이동하며 식별자 검색이 가능해야한다.
- 실행 중인 코드의 실행 순서 변경이 가능해야하며 다시 되돌아갈 수 있어야한다.

> **실행 컨텍스트** 는 식별자를 등록하고 관리하는 스코프와 코드 실행 순서 관리를 구현한 내부 메커니즘이다.

## 스택

![](/static/images/JS/executionContext/stack.jpeg)

실행컨텍스트는 스택 자료구조로 관리된다. 이를 실행컨텍스트 스택이라고 부른다.

아래의 그림은 이벤트루프, 콜스택, 이벤트큐가 작동하는 예시의 그림이다.

![](/static/images/JS/executionContext/loop.gif)

자바스크립트 엔진은 아래의 상황이 발생할때 **콜스택**에 push된다. 일반적으로 함수를 이용한 실행 컨텍스트를 사용한다.

- 전역공간은 자동으로 컨텍스트로 구성된다.
- 함수를 실행한다
- eval() 함수를 실행한다.
- block을 만든다(es6+)

```js
var a = 1 // 전역 컨텍스트

function outer() {
  // outer 컨텍스트
  function inner() {
    // inner 컨텍스트
    console.log(a) // undefined
    var a = 3
    console.log(a) // 3
  }

  inner()
  console.log(a) // 1
}

outer()
console.log(a) // 1
```

위의 코드를 실행하면 코드가 실행되는 시간에 흐름에따라 콜스택에는 push가 되고 pop이 될것이다. 이것을 한번 자세하게 다뤄보자

1. 전역코드의 평가와 실행 : 전역 코드를 평가하여 전역 실행 컨텍스트를 생성하고 실행 컨텍스트 스택에 푸시한다. 이후 전역 코드가 실행되기 시작하여 전역 변수 a 에 값이 할당되고 전역 함수 outer가 호출된다.
2. 전역 함수 outer가 호출되면 전역 코드의 실행은 일시 중단되고 코드의 제어권이 outer 함수 내부로 이동한다. 이후 엔진이 outer 함수 내부의 코드를 평가하고 outer 함수 실행 컨텍스트를 생성하고 실행 컨텍스트 스택에 푸시한다. 이때 outer 함수의 중첩함수 inner가 outer 함수 실행 컨텍스트에 등록된다. 이후 outer 함수코드가 실행되며 inner 함수가 호출된다.
3. 중첩 inner가 호출되면 outer 함수 코드의 실행은 일시 중단되고 코드의 제어권이 inner 함수 내부로 이동한다. js Engine은 inner 함수 내부의 코드를 평가하여 inner 함수 실행컨텍스를 생성하고 스택에 푸시한다. 이때 inner 함수의 지역변수 a가 inner 함수 실행컨텍스트에 등록된다. 이후 코드가 실행되며 console.log 에서 변수의 호이스팅에의해 undefined 값을 반환한후 지역 변수 a에 값이 할당된후 console.log 메서드가 호출되어 지역변수 a에 할당된 값 3이 반환된후에 inner 함수가 종료된다.
4. outer 함수로 복귀한다. 이후 inner 함수 실행컨텍스트를 실행컨텍스트 스택에서 pop 하여 제거한다. 그리고 console.log 메서드가 호출되어 지역변수 a가 없으므로 외부 실행컨텍스트인 전역실행컨텍스트에서 식별자를 조회한다. 메서드가 종료된후 outer 함수도 종료된다.
5. outer 함수가 종료되면 코드의 제어권은 다시 전역으로 이동한다. 그리고 outer 함수 실행컨텍스트를 pop 하여 제거한다. 이후 console.log 메서드가 호출되어 실행된후에 더이상 실행할 전역코드가 남아있지않아 전역실행컨텍스트도 pop 되어 실행컨텍스트 스택에는 아무것도 남아있지 않게된다.

## 렉시컬환경

식별자와 식별자에 바인딩된 값, 그리고 상위 스코프에 대한 참조를 기록하는 자료구조로 실행컨텍스트를 구성하는 컴포넌트를 뜻한다.

실행컨텍스트는 LexicalEnvironment 컴포넌트와 VariableEnvironment 컴포넌트로 구성되며 하나의 동일한 렉시컬 환경을 참조한다. 이후 몇가지 상황을 만나면 VariableEnvironment 컴포넌트를 위한 새로운 렉시컬 환경을 생성한다.

LexicalEnvironment는 EnvironmentRecord와 OuterLexicalEnvironmentReference로 구성된다.

- EnvironmentRecord : 스코프에 포함된 식별자를 등록하고 등록된 식별자에 바인딩된 값을 관리하는 저장소.
- OuterLexicalEnvironmentReference : 외부 렉시컬 환경에 대한 참조는 상위 스코프를 가리키며 이는 단방향 링크드 리스트인 스코프 체인을 구현한다.

## 전역코드 평가

소스코드가 로드되면 자바스크립트 엔진은 전역코드를 평가한다. 전역 코드 평가는 다음과 같은 순서로 진행이된다.

- 전역 실행컨텍스트 생성
- 전역 렉시컬 환경 생성
  - 전역 환경 레코드 생성
    - 객체 환경 레코드 생성
    - 선언적 환경 레코드 생성
  - this 바인딩
  - 외부 렉시컬 환경에 대한 참조 결정

![](/static/images/JS/executionContext/globalExecutionContext.png)

### 전역 실행 컨텍스트 생성

비어있는 전역 실행컨텍스트를 생성하여 실행컨텍스트에 푸시한다.

### 전역 렉시컬 환경 생성

전역 렉시컬 환경을 생성하고 전역 실행 컨텍스트에 바인딩한다.

전역 환경 레코드는 전역 변수를 관리하는 전역스코프, 전역 객체의 빌트인 전역 프로퍼티와 빌트인 전역 함수, 표준 빌트인 객체를 제공한다.

var키워드로 선언한 전역변수와, let const 키워드로 선언한 전역 변수를 구분 및 관리하기 위해 전역 스코프 역할을 하는 전역환경 레코드는 객체 환경 레코드(Object Environment Record)와 선언적 환경 레코드(Declarative Environment Record)로 구성된다.

#### 객체 환경 레코드 생성

전역 코드 평가 과정에서 var 키워드로 선언한 전역 변수와 함수 선언문으로 정의된 전역함수는 전역 환경 레코드의 객체 환경 레코드에 연결된 BindingObject를 통해 전역 객체의 프로퍼티와 메서드가 된다. 이것이 var 키워드로 선언된 전역 변수와 함수선언문으로 정의된 전역함수가 전역 객체의 프로퍼티와 메서드가 되고 전역 객체를 가리키는 식별자 없이 전역 객체의 프로퍼티를 참조하는 메커니즘이 된다.

```js
var x = 1
const y = 2

function foo() {}
```

![](/static/images/JS/executionContext/var.png)

var 키워드로 선언한 변수는 선언과 초기화 단계가 동시에 진행된다. 즉 전역 코드 평가 시점에 객체 환경 레코드에 바인딩된 BindingObject를 통해 전역 객체에 변수 식별자를 키로 등록한 다음 암묵적으로 undefined를 바인딩한다.

var키워드로 선언한 변수는 코드 실행단계에서 변수 선언문 이전에도 참조할 수 있다. 함수 선언문으로 정의한 함수가 평가되면 함수 이름과 동일한 이름의 식별자를 객체 환경 레코드에 바인딩된 BindingObject를 통해 전역 객체에 키로 등록하고 생성된 함수 객체를 즉시 할당한다.

이것이 변수호이스팅과 함수호이스팅의 차이다.

![](/static/images/JS/executionContext/declartive.png)

let const 키워드로 선언한 전역 변수는 선언적 환경 레코드(Declarative Environment Record)에 등록되고 관리된다. 이 변수들은 전역 객체의 프로퍼티가 되지않고 개념적인 블록 내에 존재한다. 여기서 개념적인 블록은 전역환경 레코드의 선언적 환경 레코드다.

### this 바인딩

전역 환경 레코드의 [[GlobalThisValue]] 내부 슬롯에 this가 바인딩된다. 일반적으로 전역 코드에서 this는 전역 객체를 가리키기땜에 전역 환경 레코드의 [[GlobalThisValue]] 내부 슬롯에는 전역 객체가 바인딩된다.

전역 환경 레코드를 구성하는 `객체 환경 레코드` `선언적 환경 레코드`에는 this 바인딩이 없다. this 바인딩은 전역 환경 레코드와 함수 환경 레코드에만 존재한다.

### 외부 렉시컬 환경에 대한 참조 결정

외부 렉시컬 환경에 대한 참조는 현재 평가중인 소스코드를 포함하는 외부 소스코드의 렉시컬 환경, 즉 상위 스코프를 가리키고 이는 단방향 링크드 리스트 스코프 체인을 구현한다.
